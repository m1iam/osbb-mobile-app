# ADR-001: Вибір технологічного стеку

## Статус
Прийнято

## Контекст
Для розробки інформаційної системи «Цифрове ОСББ» необхідно обрати технологічний стек, який забезпечить кросплатформність, зручність розробки, підтримку мобільного застосунку та можливість інтеграції із зовнішніми сервісами.

## Рішення
Для реалізації проєкту обрано такий технологічний стек:
- React Native — для розробки мобільного застосунку;
- Node.js — для серверної частини;
- MySQL — для централізованого зберігання даних;
- SQLite — для локального зберігання даних на пристрої.
- Database: MySQL (сервер) + SQLite/SQLCipher — для захисту персональних даних мешканців та цілісність результатів голосування.

## Обґрунтування
React Native дозволяє створювати один мобільний застосунок для Android та iOS, що пришвидшує розробку та зменшує витрати ресурсів команди.

Node.js є зручним середовищем для створення серверної логіки та API, а також добре підходить для роботи з мобільними застосунками та інтеграціями.

MySQL обрано як надійну реляційну систему керування базами даних для централізованого зберігання інформації про користувачів, платежі, заявки та голосування.

SQLite використовується як локальне сховище для підтримки швидкої роботи застосунку та можливого часткового офлайн-доступу.

## Наслідки
### Переваги
- єдиний код для Android та iOS;
- швидша розробка;
- зручність підтримки;
- надійне зберігання даних;
- можливість інтеграції із зовнішніми сервісами.

### Недоліки
- можливі обмеження продуктивності порівняно з нативною розробкою;
- додаткова складність при реалізації окремих нативних інтеграцій.
